# Principles

1. **Complementary operators, not unchecked failover.** Hermes and OpenClaw can review each other, but risky action stays approval-gated.
2. **Single durable brain.** gun-wiki is the shared memory layer for decisions and reusable context.
3. **Channel specialization.** Separate Hermes workspaces reduce context bleed and make outcomes easier to audit.
4. **Public-safe publication.** Share patterns and sanitized examples, never secrets or raw private logs.
5. **Promote what works.** If Hermes performs a task better than OpenClaw, move that task into the Hermes lane intentionally.

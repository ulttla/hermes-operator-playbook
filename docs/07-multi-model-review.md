# Multi-Model Review Pattern

Hermes can use a review-team pattern without depending on one specific model stack.

## Roles

- builder: proposes or implements the change
- reviewer: checks correctness and regression risk
- fact checker: verifies claims against evidence
- UX or reader reviewer: checks clarity and flow when relevant

## Output

Record agreement, disagreement, the adopted decision, and why. This creates a review trail instead of hiding model disagreement.
